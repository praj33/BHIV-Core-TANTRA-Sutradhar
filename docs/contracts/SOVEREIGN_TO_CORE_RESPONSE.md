# SOVEREIGN TO CORE RESPONSE

## Purpose

This contract defines the **decision response** that Sovereign Core sends back to BHIV Core after evaluating an ActionProposal.

---

## Decision Response Schema

```json
{
    "decision":         "string  — 'ALLOW' or 'DENY'",
    "reason":           "string  — Human-readable explanation of the decision",
    "policy_reference": "string  — Which policy was evaluated",
    "correlation_id":   "string  — The proposal_id this decision responds to",
    "trace_id":         "string  — TANTRA trace identifier (same as origin)",
    "timestamp":        "string  — ISO 8601 UTC timestamp (when decision was made)"
}
```

### Field Definitions

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `decision` | string | YES | `ALLOW` or `DENY`. No other values permitted. |
| `reason` | string | YES | Why this decision was made. For audit trail. |
| `policy_reference` | string | YES | The specific policy that was evaluated. |
| `correlation_id` | string | YES | The `proposal_id` from Core's ActionProposal. Links decision to proposal. |
| `trace_id` | string (UUID) | YES | Same `trace_id` from origin. Never generated by Sovereign. |
| `timestamp` | string (ISO 8601) | YES | When Sovereign made the decision. UTC only. |

### Example: ALLOW

```json
{
    "decision": "ALLOW",
    "reason": "Transaction analysis permitted under standard operating policy",
    "policy_reference": "sovereign.policy.standard_operations_v2",
    "correlation_id": "p-a1b2c3d4-e5f6-7890-abcd-ef0123456789",
    "trace_id": "eb823ecc-0188-4097-95c1-57667c6a9693",
    "timestamp": "2026-04-20T09:30:23.250000+00:00"
}
```

### Example: DENY

```json
{
    "decision": "DENY",
    "reason": "High-value transaction exceeds automated approval threshold",
    "policy_reference": "sovereign.policy.high_value_manual_review",
    "correlation_id": "p-a1b2c3d4-e5f6-7890-abcd-ef0123456789",
    "trace_id": "eb823ecc-0188-4097-95c1-57667c6a9693",
    "timestamp": "2026-04-20T09:30:23.250000+00:00"
}
```

---

## Core's Handling Rules

| Rule | Detail |
|------|--------|
| Core **NEVER** overrides a decision | If Sovereign says DENY, it is DENY. Period. |
| Core **NEVER** retries a decision | One proposal = one decision. No re-submission. |
| Core **NEVER** reinterprets a decision | ALLOW means ALLOW. DENY means DENY. No partial or conditional handling. |
| Core **records** the decision in trace | Decision is appended to TraceContext for auditability. |
| Core **forwards** ALLOW downstream | On ALLOW, Core passes context to Sarathi for enforcement. |
| Core **halts** on DENY | On DENY, Core records failure trace and stops. |

---

## Decision Finality

```
Sovereign Core's decision is FINAL.

Core does not:
  - Appeal
  - Retry
  - Resubmit
  - Reinterpret
  - Override

The decision flows into the trace and becomes immutable truth.
```
